出貨流程優化
傳統模式

工作站人員看單出貨

優化模式
  1. 門市銷售後同步列印
  2. 號碼單（給客戶）
  3. 揀貨單（給工作站）
  4. 揀貨單帶有取貨號碼
  5. 工作站人員對點後送出
  6. 櫃檯依號碼叫號

效益：

  • 優化客戶取貨體驗
  • 解決客訴問題

特殊銷售模式與會員管理

先開發票後出貨 (電商概念)

系統標準流程

銷售完 → 即扣庫存 → 開立發票

客戶需求

類似電商模式：

先接單 → 先開發票 → 幾天後才出貨

解決方案

系統可透過取消「完成訂單」勾選來處理

流程：

  1. 訂單已付款並開立發票
  2. 狀態為「未完成」
  3. 需手動點選「完成」才進入完成狀態

分批取貨挑戰：

若客戶付10萬發票，分批次取貨，此模式較難精確對應

此類例外流程屬於客製化處理範圍，客戶需列出詳細清單給供應商評估解法

儲值與會員點數

寄杯概念

系統已有寄杯功能

可管理已確定品項的寄杯

儲值概念

適用場景：未確定品項但先付款（如公家機關、銀行先請款）

流程：

  1. 一次性儲值（例如儲值10萬）
  2. 當下即開立發票
  3. 後續消費直接扣除會員點數
  4. 不再重複開立發票

解決問題：先開發票後分批取貨的問題

收斂複雜的付款分流，總部模組可追蹤各公司水位
